# Beta Patch Notes - v0.108.0

## Modding Updates

- First-time modded launches now copy unmodded saves into the modded save directory and show a popup explaining that modded and unmodded saves remain separate.
- Modded multiplayer version conflict errors should occur less often, and now include more detailed error popups when they do occur.
- Mega Crit still recommends using the exact same mods as your teammates, especially when those mods affect gameplay.

## Content & Balance

### General

- End-of-turn [gold]Orb[/gold] passives now trigger before [gold]Doom[/gold].
  - If a player would die to self-[gold]Doom[/gold], such as from [gold:card]Neurosurge[/gold], but an end-of-turn [gold]Orb[/gold] passive kills the last enemy first, they now win the combat instead of losing.

## Modding

- Unmodded saves are now copied to the modded save directory when launching modded for the first time.
- Added controller support for the Modding menu.
- Fixed Steam mods causing ReflectionHelper.ModTypes to return nothing.
- SavedProperty IDs from mods are now properly sorted for multiplayer.
- Content from mods with affectsGameplay set to false is sorted to the end of ID maps.
- Mods can now register multiple assemblies.
- SerializableSave.NumReloads is now consistent across peers.
- Steam mods now take precedence over local mods if the Steam version is greater.
- Batched the Steam Workshop call for checking branch support, so mods should load faster.
- Began publishing STS2.dll XML documentation.
- Fixed modded multiplayer "version mismatch" issues for players with the same mods:
  - Builds the serialization cache from ModelDb instead of via reflection.
  - Sorts content by ModelId instead of typename.
  - Uses more tiebreakers when sorting.
  - Sorts epoch IDs in `ModelIdSerializationCache`.
  - Excludes non-gameplay-affecting models from the `ModelIdSerializationCache` hash.
- Correct mod lists are now logged in the non-gameplay mod mismatch warning.
