  아래는 이 레포를 직접 읽지 않고 제미나이와 게임 아이디어 초반 구성한 내용이야 참고하도록:


너는 Slay the Spire 웹 백과사전 서비스의 프론트엔드 개발자야. 기존에 추출해 둔 슬더스 데이터(JSON)와 슬더스 2의 '점괘(Fortune) 미니게임' 애셋을 활용하여 일일 운세 미니게임을 만들려고 해. 백엔드(DB) 추가 없이 프론트엔드 자원만으로 구현하는 것이 목표야.

아래의 요구사항을 충족하는 미니게임 컴포넌트의 전체적인 구조와 핵심 로직(TypeScript + React)을 작성해 줘.

[핵심 요구사항]
1. 시드 기반 랜덤 (Daily Seed): JS 난수 생성기에 매일의 날짜(예: '2026-04-14')를 시드 값으로 사용하여, 모든 유저가 그날 같은 카드/유물 풀을 마주하게 해야 함.
2. 상태 관리: 유저의 오늘의 진행 상황(남은 골드, 획득 유물, 리롤 횟수, 최종 뽑은 3장의 점괘 카드)은 브라우저 `localStorage`에 저장하여 새로고침해도 유지되게 할 것.
3. 게임 메커니즘:
   - 매일 유저에게 기본 자원(예: 99골드) 지급.
   - 3개의 슬롯(예: 아침/점심/저녁)에 카드를 뽑아 점괘를 완성함.
   - 골드를 소모해 현재 카드를 리롤하거나, 특정 유물(예: 도박 칩, 얼어붙은 눈)의 효과를 발동해 선택에 개입할 수 있음.
4. 공유 기능: 점괘가 완성되면 결과 화면(DOM)을 `html2canvas` 라이브러리로 캡처하여 클립보드에 복사하거나 SNS에 공유할 수 있는 기능 포함.

어떻게 컴포넌트를 분리해야 할지, Daily Seed 난수 생성 함수와 localStorage를 연동하는 커스텀 훅(`useDailyRun`)은 어떻게 짜야 할지 예제 코드로 보여줘.
