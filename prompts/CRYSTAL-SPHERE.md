# 수정구 서비스 초안

## 결정

다음 신규 서비스 초안은 `수정구`로 잡는다.

이상형 월드컵은 즉시 이해되는 포맷이지만, 그대로 만들면 범용 셔플 토너먼트에
가깝다. STS2다운 차별점을 만들려면 주제 생성, 승패 맥락, 투표 집계, 밈 관리가
먼저 커진다. 반면 수정구는 이미 게임 안에 `수정구`, `소형 점술`, `대형 점술`,
`분할 납부`, 점술 대사와 이미지가 있고, 현재 백과사전의 카드/유물/포션 데이터로
바로 반복 방문형 서비스를 만들 수 있다.

## 한 줄 설명

오늘 하루의 STS2 카드 운세를 수정구가 뽑아주고, 결과의 카드/유물/포션은
백과사전 hover와 상세 페이지로 이어지는 데일리 점괘 서비스.

## 핵심 방향

- 첫 버전은 카드 중심으로 간다. 수정구의 재미는 "오늘 어떤 카드가 나왔나"가
  제일 빠르게 전달된다.
- 상호작용은 작게 넣는다. 사용자는 `소형 점술`과 `대형 점술` 중 하나만 고르고,
  결과 공개 연출을 본다.
- 유물/포션은 복잡한 조합 시스템이 아니라 `보조 징조`로 붙인다. 카드 3장에
  유물 또는 포션 1개가 더해져 결과의 톤을 바꾸는 정도가 적당하다.
- 메인 결과는 날짜 기준 공통 데일리로 생성한다. 매일 모두가 같은 주제를 보되,
  새로고침으로 무한 리롤되지 않는다.
- 개인 점괘는 보조 기능으로만 둔다. 커뮤니티 화제의 중심은 공통 데일리여야
  한다.
- 결과의 모든 카드/유물/포션은 백과사전 링크와 hover 렌더를 가진다.

## 첫 화면 경험

`/crystal-sphere`에 들어오면 설명 페이지가 아니라 바로 수정구 세션이 열린다.

1. 수정구 이미지와 짧은 게임풍 대사 표시
2. `소형 점술` 또는 `대형 점술` 선택
3. 타일 또는 카드 뒷면을 눌러 결과 공개
4. 결과 카드/유물/포션을 누르면 백과사전 상세로 이동
5. 공유 버튼으로 같은 결과를 다시 볼 수 있는 URL 생성

내비게이션 아이콘 후보:

- `public/images/sts2/events/crystal_sphere.webp`

## 모드

### 소형 점술

빠른 공유용 모드.

- 카드 1장
- 짧은 점괘 1문장
- 결과 링크 1개

예상 용도:

- "오늘의 카드"처럼 매일 한 번 확인
- SNS/커뮤니티에 가볍게 공유

### 대형 점술

서비스의 기본 모드.

- 카드 3장
- 보조 징조 1개: 유물 또는 포션
- 슬롯: `현재`, `위험`, `집어들 것`

슬롯 의미:

| 슬롯 | 역할 | 카드 해석 기준 |
| --- | --- | --- |
| 현재 | 오늘의 기본 흐름 | 카드 타입, 코스트, 캐릭터 |
| 위험 | 피해야 할 선택 | 저주/상태이상 우선, 없으면 공격적 카드 |
| 집어들 것 | 오늘 써먹을 태도 | 스킬/파워/보조 카드 우선 |
| 보조 징조 | 결과의 색 | 유물/포션 희귀도와 설명 키워드 |

## 결과 생성 규칙

시드:

- 공통 데일리: `Asia/Seoul` 기준 날짜
- 개인 점괘: 날짜 + 브라우저 프로필 id
- 공유 URL: 날짜, 모드, 결과 id를 포함한다. 공통 데일리 공유에는 프로필 id를
  포함하지 않는다.

카드 풀:

- 기본 풀은 플레이어가 실제로 볼 수 있는 카드로 제한한다.
- 토큰/상태이상/저주는 기본 슬롯에서 제외한다.
- `위험` 슬롯에서는 저주/상태이상을 별도 풀로 낮은 확률 노출할 수 있다.
- 이미지가 없는 카드나 폐기된 데이터는 제외한다.

보조 징조 풀:

- 첫 버전은 유물과 포션만 사용한다.
- 고대의 존재, 몬스터, 이벤트는 후속 확장으로 미룬다.

해석 문구:

- AI 런타임 생성 없이 게임 localization key를 차용한 fixture로 만든다.
- fixture는 `sourceTable`, `sourceKey`, `sourceText`, `serviceText`,
  `replacements`를 가진다.
- 차용 fixture는 개발자가 일방적으로 고르지 않는다. 먼저 후보 인벤토리를 만들고,
  후보별로 include/exclude를 사용자 큐레이션으로 확정한다.
- 예: `events.CRYSTAL_SPHERE.banter.REVEAL_GOOD.3`의
  `다정한 행운이 당신에게 깃들기를!`를 차용해 `다정한 {entity}이(가) 당신에게
  깃들기를!` 같은 서비스 문구로 변형한다.
- 예: `events.THE_LANTERN_KEY.pages.INITIAL.description`의 열쇠 대사를
  역사 강의서에서 차용한 것처럼, 수정구도 이벤트/죽음/보상/선택 문구를 source
  key 기준으로 관리한다.
- 카드 이름과 게임 설명은 게임 localization을 그대로 사용한다.
- 점괘 문장은 서비스 문구이므로 한국어를 canonical로 두고 영어 보조 번역을
  추가한다.

## STS2다운 포인트

- 게임의 `수정구` 이벤트 명칭과 점술 버튼명을 그대로 활용한다.
- 공개 연출은 수정구 이벤트의 "타일을 열어 미래를 드러낸다" 구조를 빌린다.
- 점괘 문장은 자유 작문이 아니라 실제 게임 문구의 source key를 드러내고
  비틀어 쓴다.
- 결과가 단순 텍스트가 아니라 실제 카드/유물/포션 렌더와 연결된다.
- `분할 납부`는 후속 기능으로 남긴다. 예를 들어 저주 카드를 하나 더 공개하고,
  대신 보조 징조를 하나 더 받는 선택지로 확장할 수 있다.

## 문구 큐레이션 흐름

차용 문구는 후보 전체를 먼저 볼 수 있어야 한다.

1. 로컬 게임 localization에서 후보 문구를 추출한다.
2. 후보마다 `sourceTable`, `sourceKey`, 한국어 원문, 영어 원문, 카테고리,
   차용 아이디어, 상태를 붙인다.
3. 상태는 `candidate`, `include`, `exclude`, `needs-edit`로 관리한다.
4. 사용자는 후보를 하나씩 보며 채택 여부를 결정한다.
5. `include`가 된 문구만 수정구 fixture 데이터로 승격한다.

초기 후보 카테고리:

| 카테고리 | 예시 source key | 용도 |
| --- | --- | --- |
| 수정구 점술 | `CRYSTAL_SPHERE.banter.*` | 세션 시작, 공개, 종료 |
| 이벤트 선택 | `*.pages.*.options.*` | 선택지형 점괘, 분할 납부 확장 |
| 이벤트 대사 | `*.pages.*.description` | 소제목, 결과 설명 |
| 실패/죽음 | `*.loss`, `game_over_screen.*` | 위험 슬롯, 저주/상태이상 |
| 전투/보상 UI | `card_reward_ui.*`, `combat_messages.*` | 카드 보상 판단, 오늘의 질문 |

## MVP 범위

- `/crystal-sphere` 라우트
- 서비스 문구: 한국어/영어
- 게임 문구: 현재 `gameLocale` 기준 카드/유물/포션 이름과 설명
- 소형 점술
- 대형 점술
- 결정론적 daily seed
- 게임 문구 차용 후보 인벤토리
- 사용자 큐레이션을 거친 게임 문구 차용 fixture
- 결과 공유 URL
- 백과사전 hover/link
- 모바일 우선 화면

## 제외

- 서버 저장
- 랭킹
- 댓글
- 개인별 결과만 있는 운세 서비스
- 몬스터/이벤트/고대의 존재까지 섞는 cross-resource 점괘
- 복잡한 점수 계산이나 궁합 계산

## 후속 확장

1. `분할 납부`: 저주/상태이상을 감수하고 추가 점괘를 공개한다.
2. `대형 점술` 보드: 3장 고정이 아니라 작은 타일 보드에서 몇 칸을 여는 방식.
3. 캐릭터별 수정구: 아이언클래드/사일런트/디펙트/네크로바인더/리젠트 필터.
4. 오늘의 공유 피드: Supabase가 안정화된 뒤 익명 공유 결과만 모아 보여준다.
5. 패치 버전 고정: 특정 STS2 버전 기준으로 점괘를 다시 볼 수 있게 한다.

## 수용 기준

- 새로고침해도 같은 날짜의 공통 데일리는 같은 결과를 본다.
- 공유 URL은 다른 브라우저에서도 같은 결과를 재현한다.
- 결과에 나온 카드/유물/포션은 모두 백과사전 상세로 이동한다.
- 게임에서 온 이름과 설명은 직접 번역하지 않는다.
- 차용 문구는 source localization key와 원문을 추적한다.
- fixture로 쓰는 차용 문구는 사용자 큐레이션 상태를 가진다.
- 첫 화면에서 사용자는 별도 설명을 읽지 않아도 점술을 시작할 수 있다.

## 남은 질문

1. 개인 점괘를 MVP에 같이 넣을 것인가, 아니면 공통 데일리만 먼저 만들 것인가?
   - 추천: 공통 데일리만 먼저 만든다. 개인 점괘는 공유/댓글 반응을 본 뒤 붙여도
     된다.
2. 소형 점술을 기본으로 둘 것인가, 대형 점술을 기본으로 둘 것인가?
   - 추천: 대형 점술을 기본으로 둔다. 서비스의 차별점은 카드 3장과 보조 징조에서
     나온다.
3. `분할 납부`를 MVP에 넣을 것인가?
   - 추천: 넣지 않는다. 이름은 매력적이지만 첫 버전에서는 저주/추가 보상의 의미가
     과하게 커진다.
