# i18n 계획

슬서운이야기의 i18n은 하나의 번역 테이블이 아니라, 문구의 출처에 따라
세 계층으로 나눈다. 한국어는 서비스의 일급시민 언어이고, 영어는 서비스
UI의 두 번째 지원 언어다. 게임 내 문구는 서비스 언어와 독립적으로 게임이
지원하는 14개 언어를 그대로 따른다.

## 언어 축

### 서비스 언어

`serviceLocale`은 서비스가 직접 소유한 UI 문구의 언어다.

- `ko` — 기본값이자 canonical 경로
- `en` — 보조 서비스 언어

한국어 일급시민 정책을 위해 URL은 한국어를 prefix 없는 기본 경로로 둔다.
영어 UI는 `/en` prefix를 붙인다.

| 서비스 언어 | URL 예시 |
| --- | --- |
| 한국어 | `/codex/cards`, `/patches` |
| 영어 | `/en/codex/cards`, `/en/patches` |

이 방식은 기존 한국어 URL과 검색 색인을 유지하고, 영어를 명시적인 보조
경로로 분리한다. `/ko` prefix는 만들지 않는다.

### 게임 문구 언어

`gameLocale`은 게임 파일에서 온 카드, 유물, 포션, 파워, 키워드, 이벤트,
몬스터, 전투, 인게임 UI 문구의 언어다.

지원 언어는 `data/sts2/languages.json`의 14개 언어와 동일하다.

- `deu`, `eng`, `esp`, `fra`, `ita`, `jpn`, `kor`
- `pol`, `ptb`, `rus`, `spa`, `tha`, `tur`, `zhs`

기본값은 서비스 언어를 따라간다.

| serviceLocale | default gameLocale |
| --- | --- |
| `ko` | `kor` |
| `en` | `eng` |

사용자가 게임 문구 언어를 바꾸면 URL 상태로 보존한다. 예를 들어
`/en/codex/cards?gl=zhs`는 영어 UI에 중국어 게임 문구를 표시한다. 한국어
게임 문구가 아닌 game-only locale은 영어 서비스 UI로 canonicalize한다. 이
URL 상태는 다른 사람에게 링크를 공유했을 때도 같은 조합을 재현하기 위한
것이다.

### 접속 시 자동 언어 선택

명시적인 URL 상태가 없는 첫 접속은 요청 신호를 바탕으로 locale을 고른다.

우선순위:

1. URL에 `gl`이 있으면 그 값을 사용하고, 해당 `gameLocale`에 맞는
   `serviceLocale` 경로로 canonicalize한다.
2. `/en` prefix가 있으면 영어 서비스 UI + 영어 게임 문구를 사용한다.
3. 이전 선택이 쿠키에 저장되어 있으면 그 값을 사용한다.
4. 쿠키가 없으면 `Accept-Language`를 먼저 보고, 없거나 지원 언어를 찾지
   못하면 Vercel/프록시의 국가 헤더(`x-vercel-ip-country`, `cf-ipcountry`
   등)를 본다.
5. 아무 신호도 없으면 한국어 서비스 UI + 한국어 게임 문구를 기본값으로 둔다.

브라우저/OS locale은 일반적으로 `Accept-Language`와 `navigator.languages`에
반영된다. 서버 첫 요청에서는 `Accept-Language`와 국가 헤더만 사용할 수
있으므로, 언어 드롭다운에서 사용자가 고른 값은 쿠키에 저장해 다음 접속의
기준으로 삼는다. 공유 URL의 명시적인 `gl` 값은 쿠키나 자동 감지가 덮어쓰지
않는다.

## 문구 소유권

### 1. 서비스 문구

서비스 문구는 슬서운이야기가 직접 작성하고 번역한다.

예시:

- 내비게이션: 패치노트, 백과사전, 역사 강의서
- 검색 placeholder, 필터 제목, 빈 상태, 버튼, 탭, 정렬 tooltip
- 댓글 UI의 껍데기: 입력 placeholder, 로딩, 삭제, 에러
- 페이지 metadata와 Open Graph 문구
- 패치노트 페이지의 서비스 프레임

구현 원칙:

- `src/messages/service.ts` 같은 typed dictionary로 관리한다.
- React 컴포넌트 안의 새 하드코딩 서비스 문구를 금지한다.
- 한국어 문구를 source of truth로 보고 영어는 서비스 번역으로 관리한다.

### 2. 게임 문구

게임 문구는 번역하지 않는다. 게임 파일의 localization을 그대로 추출해
사용한다. 오타, 어색한 번역, 원문 영어 변경, 패치로 바뀐 문구도 그대로
따른다.

예시:

- 카드/유물/포션/파워/인챈트/키워드 이름과 설명
- 이벤트 이름, 페이지, 선택지, 대사
- 몬스터, 전투, 행동 이름
- 희귀도, 카드 유형, 캐릭터명처럼 게임에서 온 분류명
- 패치노트 rich text에서 hover/link 대상이 되는 모든 게임 요소

구현 원칙:

- SSOT는 Steam 패치노트와 로컬 게임 PCK/DLL이다.
- `spire-codex.com` 데이터는 SSOT로 사용하지 않는다.
- 표시명으로 링크 대상을 찾지 않는다. 장기적으로 `[entity:card:ID]`처럼
  ID 기반 마크업을 사용한다.
- `src/lib/codex-types.ts`의 한국어 문자열 union은 canonical id 기반 타입으로
  줄이고, 표시 label은 `gameLocale`별 localization에서 가져온다.
- `src/components/codex/codex-description.tsx`의 수동 keyword description은
  PCK 기반 데이터로 대체한다.

### 3. 번역하지 않는 커뮤니티/밈 문구

다음 컨텐츠는 서비스 언어가 바뀌어도 번역하거나 현지화하지 않는다.

- 댓글 본문
- 슬서운 이야기 문장
- 케미컬X 게시글과 사용자 정의 키워드
- 커뮤니티가 만든 밈, 말장난, 제보 문구

이 영역의 UI 껍데기는 서비스 문구이므로 `serviceLocale`을 따른다. 본문은
작성된 원문 그대로 보여준다. 한국어 컨텐츠를 영어 UI에서 봐도 한국어
본문으로 남고, 영어 컨텐츠를 한국어 UI에서 봐도 영어 본문으로 남는다.

### 4. 프로필 식별 문구

프로필은 자격증명, 인증, 인가가 아니다. Supabase RLS가 댓글/케미컬X 작성자를
대충 식별하기 위한 휘발성 식별 속성 묶음이며, 개인정보나 영구 계정성을
전제로 하지 않는다. 다른 기기로 옮기지 못하거나 초기화되어도 서비스 의미가
깨지면 안 된다.

프로필 화면은 세 종류의 문구가 섞인다.

| 화면 요소 | 소유권 | `serviceLocale` 영향 | `gameLocale` 영향 | 저장/표시 정책 |
| --- | --- | --- | --- | --- |
| 프로필 화면의 버튼, 행 제목, action label, aria label, metadata | service-only | `ko`/`en` dictionary | 없음 | 새 하드코딩 금지 |
| 캐릭터/펫/고대신의 실제 이름, 고대신 epithet | exact game text | 없음 | 선택된 게임 언어 표시명 | PCK/추출 데이터 기준 |
| 프로필 닉네임 | localized community fixture at creation | 새로 생성할 때 locale별 fixture 선택 | 없음 | 한 번 선택/저장된 닉네임 문자열은 자동 번역하지 않음 |
| 댓글/케미컬X 작성자명으로 표시되는 프로필 닉네임 | no-i18n user identity string | UI 껍데기만 영향 | 없음 | 저장된 원문 그대로 표시 |

프로필 닉네임은 일반 커뮤니티/밈 본문과 달리 서비스가 제공하는 선택 fixture다.
따라서 `serviceLocale`별 fixture를 둘 수 있다. 단, 사용자가 선택했거나 RLS
식별자와 함께 저장된 뒤에는 사용자 정체성 문자열이 되므로, locale 변경 시
자동 번역하거나 다른 fixture로 치환하지 않는다. 새 닉네임을 생성할 때만 현재
`serviceLocale`의 fixture pool을 사용한다.

한국어 fixture는 한국 커뮤니티 밈을 source로 삼고, 영어 fixture는 Reddit 등
영어권 커뮤니티에서 실제로 쓰이는 축약명/애칭을 우선하되 불확실하면 공식
영문명을 fallback으로 둔다. 현재 프로필 mock의 초기 fixture는 다음과 같다.

| 캐릭터 | `ko` fixture | `en` fixture |
| --- | --- | --- |
| 아이언클래드 | 아클단, 아평, 아이언클래스, 아이언클레임, 아이돌클라스, 아장연 | Clad, The Clad, Ironclad |
| 사일런트 | 사일단, 사평, 사장연 | Silent, The Silent, Shiv Silent |
| 리젠트 | 리황, 리평 | Regent, Reggie, King Reggie |
| 네크로바인더 | 네바, 네크로맨서, 네평, 골골맘, 네크단 | Necro, Necrobinder, Necro Binder |
| 디펙트 | 디평, 디펙터, 디황 | Defect, The Defect, Orb Defect |

프로필 UI에서 쓰는 서비스 문구는 아래 id로 dictionary에 올린다. 같은 한국어
단어라도 Codex 카드 유형/몬스터 행동명 등 게임 문구로 쓰일 때는 이 표를
재사용하지 않는다.

| --- | --- | --- | --- |

## 차용 게임 문구

서비스 문구 중에는 게임 문구를 패러디하거나 일부 치환한 문장이 있다.

예:

- 게임 원문: `그 열쇠를 찾으러 사방팔방 뒤지고 있었다네! 혹시 실례가 아니라면…`
- 서비스 문구: `그 도전 이력을 찾으러 사방팔방 뒤지고 있었다네! 혹시 실례가 아니라면…`

이 문구는 임의 번역으로 자동 전환하지 않는다. 대신 `borrowedGamePhrase`
fixture로 관리한다. 단, 원문과 치환어가 모두 PCK에 있는 경우에는
`gameLocale`별 원문을 읽고 PCK 치환어를 끼워 넣는 `gameLocaleRuntime`
방식을 사용할 수 있다.

fixture는 다음 정보를 갖는다.

- `id` — 서비스 문구 id
- `serviceLocale` — 이 문구가 속한 서비스 언어
- `sourceLang` — 원본 게임 문구 언어
- `sourceTable` — PCK localization table
- `sourceKey` — PCK key
- `sourceText` — 기대하는 게임 원문
- `serviceText` — 서비스에서 실제 쓰는 차용 문구
- `replacementTable` / `replacementKey` — `gameLocaleRuntime`에서 치환어를
  가져오는 PCK table/key
- `replacements` — 어떤 부분을 의도적으로 바꿨는지

검증은 원본 게임 문구나 치환어가 패치로 바뀌었는지 감지하고, 차용 문구를
재검토하게 만드는 역할만 한다. 자동 번역이나 자동 수정은 하지 않는다.

## UI/UX 원칙

- 기본 화면은 한국어 서비스 UI + 한국어 게임 문구다.
- 영어 서비스 UI에서는 영어 게임 문구가 기본이다.
- 게임 문구 언어가 서비스 언어와 다를 때는 언어 선택 UI에 `简体中文 (게임
  텍스트만)`처럼 범위를 분명히 표시한다.
- 영어 서비스에서 `gameLocale=eng`일 때 패치노트 개발자 코멘트 원문을 별도
  중복 노출하지 않는다.
- 한국어 서비스에서 개발자 코멘트는 한국어 번역을 먼저 보여주고, 영어 원문은
  접힘/토글로 확인할 수 있게 한다.

### 폰트 소유권

폰트도 문구 소유권과 같은 기준으로 나눈다. Tailwind의 `font-sans` 같은
일반 토큰을 서비스 폰트로 재정의하지 않는다.

- 서비스 UI 문구는 `font-service`를 사용한다.
- 게임에서 온 제목, 이름, 분류명은 `font-game-title`을 사용한다.
- 게임에서 온 본문, 설명, hover tip, 카드 설명은 `font-game-text`를 사용한다.
- 런타임은 `<html data-service-locale data-game-locale>` 값으로 게임 폰트
  CSS 변수를 전환한다.
- 한국어 게임 문구는 게임에서 쓰는 한국어 폰트로 fallback되어야 하고, 영어
  게임 문구는 게임에서 쓰는 영어 폰트로 표시되어야 한다.
- 커뮤니티/밈 본문은 작성된 원문의 성격을 유지하되, UI 껍데기만 서비스
  폰트를 따른다.

## 패치노트 i18n/l10n 진리표

패치노트는 Steam 원문을 서비스가 재구성한 컨텐츠지만, 그 안에서 게임에서 온
명칭은 모두 `gameLocale`을 따른다. `serviceLocale`은 패치노트 파일 선택과
서비스 껍데기만 결정한다.

| 화면 요소 | 소유권 | source | `serviceLocale` 영향 | `gameLocale` 영향 | fallback |
| --- | --- | --- | --- | --- | --- |
| 패치 목록/상세의 버튼, 날짜 주변 라벨, 댓글 제목 | service-only | `src/messages`/페이지 copy | `ko`/`en` | 없음 | 서비스 기본값 |
| 패치노트 본문 문장 | authored patch content | `data/sts2-patch-notes/*.ko.md` 또는 `*.md` | 파일 선택 | 없음 | 반대 서비스 언어 파일 |
| `[devnote]` 한국어 번역 | authored patch content | `*.ko.md` | 한국어 서비스에서 노출 | 없음 | 없음 |
| `[devnote:en]` 원문 | source quote | Steam 원문 | 한국어 서비스에서 접힘 원문 | 없음 | 없음 |
| `[gold:type]...[/gold]` 게임 엔티티 표시명 | exact game text | Codex entity + PCK localization | 링크/툴팁 UI만 영향 | 선택된 게임 언어 표시명 | `kor`에서는 한국어 structured data, 그 외에는 영어 structured data |
| `**Name**` bold 게임 엔티티/키워드 | exact/derived game text | PCK localization title/name table | 없음 | 선택된 게임 언어 표시명 | 링크 가능한 엔티티가 없으면 노란 bold 텍스트 |
| 게임 유래 섹션 제목, 예: `Colorless Cards` | derived game text | PCK key에서 안전 파생 (`card_library.POOL_COLORLESS_TIP`) | 일반 섹션 제목은 서비스 파일 언어 유지 | 게임 유래 제목만 선택된 게임 언어 | 파생 불가 시 원문 유지 |
| 일반 섹션 제목, 예: `Highlights`, `Bug Fixes`, `Audio` | authored patch structure | 패치노트 파일 | 파일 선택 | 없음 | 원문 유지 |
| 댓글/사용자 컨텐츠 | no-i18n | 사용자 입력 | 껍데기 UI만 영향 | 링크/툴팁 매칭 후보만 영향 | 본문은 작성된 원문 유지 |

패치노트 rich 렌더러는 다음 순서를 지킨다.

1. 타입 힌트가 있으면 (`[gold:card]`, `[gold:monster]` 등) 해당 타입에서만
   엔티티를 찾는다.
2. 타입 힌트가 없고 같은 이름이 여러 타입에 있으면, `epoch`처럼 보조적인
   링크 대상보다 카드/유물/몬스터/고대의 존재 같은 직접 게임 요소를 우선한다.
3. 링크 가능한 엔티티가 없더라도 PCK title/name 테이블에 있는 게임 용어라면
   `gameLocale` 표시명으로 바꿔 노란 bold 텍스트로 표시한다.
4. PCK에서 제거되어 선택 언어 localization key가 없어진 과거 엔티티는
   `gameLocale=kor`에서는 한국어 structured data, 그 외에는 영어 structured
   data로 fallback한다. 비한국어 게임 언어에서 한국어 fallback을 노출하지 않는다.
5. 영어 패치노트 원문에 평문으로 남은 `(buff)`, `(nerf)`,
   `(revert/buff)` 같은 변경 분류는 한국어 패치노트와 같은 green+sine,
   red+jitter 효과로 렌더한다.

댓글은 작성자가 입력한 본문 표시명을 번역하지 않는다. 다만 현재 화면의
`gameLocale` 표시명뿐 아니라 한국어/영어 기본 표시명과 alias도 링크 후보로
사용해, 한국어로 작성한 `목 조르기` 댓글이 영어 only game locale 화면에서도
`목 조르기` 그대로 gold 키워드/툴팁으로 렌더되게 한다.

v0.105.0처럼 Steam 원문에 “언어별 번역 추가/오역 수정”이 포함된 패치 이후에는
패치노트 작업 전에 PCK localization dump와 `data/sts2/localization/*` 동기화를
확인하고, rich 렌더러의 keyword/title label map이 `badges`, `cards`,
`encounters`, `monsters`, `epochs`, `ancients` 등 패치에서 언급된 table을
포함하는지 점검한다.

## 구현 순서

1. 이 문서와 fixture 정책을 먼저 고정한다.
2. PCK에서 14개 언어 localization을 추출하는 스크립트를 추가한다.
3. 서비스 문구 dictionary를 추가하고 신규 UI 문구부터 dictionary를 쓰게 한다.
4. codex 데이터 로더를 `gameLocale` 인자로 확장한다.
5. 한국어 문자열 기반 타입/필터를 canonical id 기반으로 점진 교체한다.
6. rich patch note 마크업을 표시명 기반에서 ID 기반으로 전환한다.
7. CI에 i18n 검증을 추가한다.

## 회귀 방지

필수 검증:

- 14개 게임 언어의 핵심 table이 같은 ID 집합을 갖는지 확인
- `gameLocale` fallback 누락 확인
- 차용 게임 문구 fixture의 `sourceText`가 현재 PCK와 일치하는지 확인
- 서비스 문구 dictionary 밖의 새 하드코딩 문구 탐지
- 기존 `pnpm lint`, `pnpm build`
