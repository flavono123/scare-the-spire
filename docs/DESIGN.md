## Compendium Bestiary UI

### 확정된 기준

- 게임 구현 기준: `NBestiary.AddAct()`는 막 구분자를 먼저 추가하고, 각 막 안에서 일반 전투 → 엘리트 → 보스 → 이름순으로 정렬한다.
- 게임 색상 기준: `NBestiaryActDivider`는 막 제목을 파란색으로 표시한다. `NBestiaryEntry`는 일반 몬스터를 cream/white, 엘리트를 purple, 보스를 red로 표시한다.
- 몬스터 도감 상세의 행동은 시작점이 있는 상태 전이표로 보여준다. 명시적인 `__START__` 전이가 있으면 그대로 쓰고, 없으면 `move_graph.initial`을 시작점으로 보강한다.
- 행동 미리보기 목록은 게임의 `MonsterModel.GenerateBestiaryMoveList()` 기준을 따른다. 턴 행동 외에 스켈레톤에 정확한 `revive`, `hurt`, `die` 애니메이션이 있으면 별도 미리보기 행동으로 노출한다.

### 아직 결정할 것

- 인덱스 레이아웃: 게임처럼 긴 가로 스크롤 이름 목록으로 갈지, 현재처럼 이미지와 이름 중심의 정적/idle Spine 목록으로 갈지 결정이 필요하다. 당장은 장황한 HP, 등장 막, 행동 수 정보는 인덱스 타일에서 제거한다.
- 상세 레이아웃: 현재 카드 컨테이너 위주의 구조를 줄이고, Spine 렌더가 배경에 떠 있는 듯한 비스티어리식 화면으로 옮길지 별도 시안이 필요하다. 이 결정은 몬스터 도감뿐 아니라 전체 Compendium 화면 체계와 같이 맞춘다.
- 인카운터 연결: 몬스터 상세의 등장 전투를 인카운터 상세 페이지로 직접 이동 가능하게 하고, 인카운터 상세에서도 몬스터 상세로 이어지는 양방향 탐색을 강화한다.
- 인카운터 정밀성: 현재 `scripts/parse-encounters.py`는 encounter 파일의 `ModelDb.Monster<T>()` 참조를 스캔해 가능한 몬스터를 평탄화한다. 예를 들어 `SlitheringStranglerNormal.GenerateMonsters()`는 `잭스프루트 1마리`, `중형 슬라임 1마리`, `소형 슬라임 2마리` 중 하나와 교살마를 조합하지만, 현재 JSON은 모든 후보를 한 목록에 담는다. 실제 조합을 보여주려면 `GenerateMonsters()`의 선택 분기와 확률/동등 선택 여부를 별도 구조로 추출해야 한다.
- 인카운터 패턴 조합: 정밀 조합이 확보되면 각 몬스터의 `move_graph`를 조합해 "동시에 공격 가능한 턴" 같은 요약을 만들 수 있다. 다만 전투 중 처치, 기절, 소환, 상태 변화로 실제 턴은 달라지므로 UI에는 "초기 조합 기준 예상"으로 표시해야 한다.
- Spine/VFX 표시 옵션: 미완성, mimic, static-only, 파싱 실패 VFX는 나중에 토글 가능한 메타데이터로 분리한다. 패치별 VFX 버전 표시는 기본 요구사항이 아니며, 최신 PCK 상태를 Compendium 렌더 기준으로 삼는다.
