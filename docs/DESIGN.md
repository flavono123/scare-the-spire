# 디자인 가이드

슬서운이야기는 슬레이 더 스파이어 2 패치와 게임 데이터에서 출발하는 정보 사이트다. STS2 백과사전(Compendium)이 현재 서비스의 중심이며, 화면은 게임 위키보다 게임 안 백과사전에 가까운 밀도와 질감을 우선한다.

## 정체성

- 한국어를 일급 시민 언어로 둔다.
- 게임에서 유래한 이름과 용어는 공식 번역을 우선한다.
- 데이터가 중심이다. 추측성 장식이나 서비스 편의로 만든 값보다 게임 원본/PCK/Steam 패치노트에서 확인된 정보를 우선한다.
- "엔티티"라는 표현은 UI에 쓰지 않는다. 카드, 유물, 포션, 몬스터, 이벤트, 고대의 존재처럼 직접 부른다.
- 첫 화면은 실제 탐색/검색/비교가 가능한 도구여야 한다. 설명형 랜딩 페이지나 큰 마케팅 히어로는 만들지 않는다.

## 전체 톤

다크 모드 전용의 어두운 판타지 UI를 유지한다. 화면은 조용하고 정보 밀도가 높아야 하며, 반복 탐색에 방해되는 과한 장식, 커다란 카드 섹션, 원노트 그라디언트 배경을 피한다.

| 용도 | 값 | 비고 |
| --- | --- | --- |
| 페이지 배경 | `bg-background`, `#0f0f13` 계열 | 거의 검정에 가까운 남색 |
| 패널/상단바 | `#16162a`, `#1a1a2e` | 백과사전 공통 셸 |
| 주요 텍스트 | `#e4e4e7`, `zinc-200` | |
| 보조 텍스트 | `#a1a1aa`, `zinc-400` | |
| 약한 텍스트 | `zinc-600` | 카운트, 영어 보조명 |
| 기본 강조 | `#eab308`, `yellow-500` | 카드/유물 등 게임 요소 강조 |
| 버프 | `green-500` + sine | rich text 기준 |
| 너프 | `red-500` + jitter | rich text 기준 |
| 고대의 존재 카테고리 | blue | 특정 고대의 존재 링크는 gold 가능 |
| 백과사전 막 텍스트 | `#60a5fa`, `text-blue-300` | 몬스터 도감과 동일하게 모든 실제 막에 적용 |
| 막 무관 | `#a1a1aa`, `text-zinc-400` | 실제 막과 구분 |

## 컨테이너와 선택 컨트롤

반복 선택 UI에서 아이템을 담기 위해 필요한 stretch wrapper, grid cell, carousel item은 되도록 컨테이너처럼 보이지 않게 한다.

- 선택 아이콘은 배경 위에 떠 있는 게임 아이콘처럼 보여야 한다. 기본 상태의 카드형 배경, 테두리, 패널 그림자는 피한다.
- 선택 상태는 외곽 박스보다 밝기, 크기, glow, 작은 표시점처럼 아이콘 자체에 가까운 신호로 표현한다.
- 아이콘 아래 고정 라벨은 공간을 늘리고 카드처럼 보이게 만들면 쓰지 않는다. 이름과 보조 정보는 `title`/tooltip 같은 hover affordance로 제공한다.
- 실제 정보를 구획하는 패널이 아니라 정렬만을 위한 컨테이너라면 배경과 경계를 없앤다.
- 우측 preview/render 영역은 여러 칸으로 쪼개기보다 하나의 큰 무대처럼 쓰고, 선택 목록은 필요한 폭만 차지한다.

## 게임 충실도와 웹 차이

게임 원본을 따르는 대상과 웹 서비스라 달라지는 대상을 분리한다.

- 게임 데이터, 공식 명칭/번역, 추출 에셋, 카드/유물/포션/몬스터 같은 리소스의 시각적 정체성은 게임 원본을 우선한다.
- 검색, 필터, URL 공유, 댓글, 좋아요, 패치노트 hover/link, 비교/정렬처럼 게임에 없는 기능은 웹 탐색과 공유를 위해 추가할 수 있다.
- 웹 기능을 추가하더라도 게임 리소스 자체를 일반 카드형 패널 안에 가두지 않는다. 게임에서 경계 없이 떠 있는 요소는 웹에서도 배경 위에 떠 있는 무대처럼 보여준다.
- 게임에 대응되는 컨테이너가 없는 추가 정보는 게임 에셋보다 덜 두드러지는 보조 패널, 하단 영역, 사이드 영역으로 분리한다.
- 게임과 다른 표현은 “웹에서 찾고 비교하고 공유하기 위해서”만 허용하며, 새 장식 체계나 임의의 판타지 UI를 만들기 위한 이유로 쓰지 않는다.

## 백과사전 공통 레이아웃

카드, 유물, 포션, 파워, 인챈트, 몬스터, 인카운터, 이벤트, 고대의 존재 목록은 가능한 한 백과사전 공통 셸과 상단바를 공유한다.

- 좌측 필터 드로어는 데스크톱에서 고정 폭으로 열리고, 모바일에서는 필터 버튼으로 열리는 drawer로 동작한다.
- 상단바는 필터 토글, 짧은 제목, 중앙 검색창, 결과 수, 필요 시 버전 선택기를 포함한다.
- 목록 본문은 상단바 아래에서 독립 스크롤된다.
- 큰 페이지 제목 블록은 백과사전 목록 화면에 쓰지 않는다.
- 검색창은 각 화면의 주요 탐색 도구이며 `Cmd/Ctrl+K`로 포커스한다.
- 필터 trigger 표기는 검색 토큰과 맞춘다: `@` 캐릭터/소속, `#` 유형, `$` 희귀도, `!` 비용, `%` 막.

## 색상 의미

### 캐릭터

캐릭터 색상은 타이틀, 섹션, 배지, 필터 아이콘 보강에 제한적으로 쓴다.

- 아이언클래드: red
- 사일런트: green
- 리젠트: orange
- 네크로바인더: pink
- 디펙트: aqua

### 막

백과사전에서 막은 지리/진행 구분이지 캐릭터나 희귀도 분류가 아니다. 실제 막 이름은 몬스터 도감의 `NBestiaryActDivider` 감각에 맞춰 항상 파란색 텍스트를 사용한다.

- 실제 막 텍스트: `text-blue-300`
- 실제 막 점/섹션 색: `#60a5fa`
- 막 무관: `text-zinc-400`, 점은 `#666`
- 이벤트/고대의 존재/인카운터에서도 막별 green/orange/yellow 색을 쓰지 않는다.

## 목록 화면

### 카드

- 카드 이미지를 중심으로 grid를 구성한다.
- 캐릭터, 유형, 희귀도, 비용, 이름 정렬을 좌측 필터에서 제공한다.
- 기본/일반/고급/희귀 외 특수 카드군은 "기타" 하위 필터로 정리한다.
- 업그레이드, 베타 아트, 멀티플레이어 카드, 댓글/좋아요 표시는 명시 토글로 둔다.

### 유물

- 작은 유물 아이콘 grid를 유지한다.
- 희귀도 단위로 섹션을 나누고, 고대 유물은 고대의 존재별 하위 그룹을 보여준다.
- 이벤트 유물은 소속 필터에서 별도로 고를 수 있어야 한다.

### 포션

- 포션 아이콘 중심 grid를 유지한다.
- 일반/고급/희귀/특별 섹션으로 묶는다.
- 캐릭터/공용/이벤트 pool 필터를 제공한다.

### 파워와 인챈트

- 파워는 Buff/Debuff/Other처럼 게임 내 역할을 기준으로 묶는다.
- 인챈트는 적용 카드 유형과 중첩 가능 여부가 핵심 필터다.
- 설명 텍스트에는 rich description 렌더링과 cross-reference를 유지한다.

### 몬스터

- 게임 구현 기준: `NBestiary.AddAct()`처럼 막 구분자를 먼저 두고, 각 막 안에서 일반 전투, 엘리트, 보스, 이름순으로 정렬한다.
- 막 제목은 `#60a5fa` 파란색으로 표시한다.
- 몬스터 이름/유형은 게임의 `NBestiaryEntry` 감각에 맞춰 일반 몬스터 cream/white, 엘리트 purple, 보스 red를 유지한다.
- 상세 화면의 행동은 시작점이 있는 상태 전이표로 보여준다. 명시적인 `__START__` 전이가 있으면 그대로 쓰고, 없으면 `move_graph.initial`을 시작점으로 보강한다.
- 행동 미리보기 목록은 게임의 `MonsterModel.GenerateBestiaryMoveList()` 기준을 따른다.

### 인카운터

- 막과 전투 유형이 핵심 필터다.
- 몬스터 상세와 인카운터 상세은 양방향 탐색 가능해야 한다.
- encounter 원본의 조합 분기가 아직 평탄화된 경우, UI에서 확정 조합처럼 표현하지 않는다.

### 이벤트

- 백과사전 공통 셸을 사용한다.
- 좌측 막 필터와 섹션 제목은 몬스터 도감과 같은 파란 막 스타일을 쓴다.
- 목록 항목은 이벤트 이미지를 낮은 대비 배경으로 깔고, 한국어 이름과 영어 원문을 함께 표시한다.
- 클릭 시 상세 모달을 열고, URL query에 선택 이벤트를 반영한다.

### 고대의 존재

- 백과사전 공통 셸을 사용한다.
- 막 필터와 섹션 제목은 이벤트와 동일한 파란 막 스타일을 쓴다.
- 카드는 고대의 존재 이미지, 이름, 영어명, 이명, 등장 막, 보상 유물 수를 보여준다.
- 고대의 존재 자체는 blue 계열 정체성을 유지하되, 개별 리소스 링크/강조가 필요할 때는 rich text의 gold 링크 규칙을 따른다.

## 상세 화면

- 항목의 상세는 모달과 상세 페이지로 깊이를 나누지 않는 하나의 상세 표면이다.
- 목록에서 항목을 클릭하면 대체로 모달 표현을 우선하고, 직접 URL로 들어오면 같은 상세 표면을 주소 가능한 페이지로 렌더한다.
- 모달은 웹 서비스의 탐색 기능이지만, 상세 표면 안의 주 리소스 표현은 게임식 화면을 우선한다.
- 카드 도서관처럼 게임에서 카드가 컨테이너나 경계선 없이 떠 있는 경우, 카드 상세도 카드 자체를 배경 위에 띄우고 불필요한 카드형 wrapper를 만들지 않는다.
- 상세 표면은 배경 dim + blur, Escape, 바깥 클릭 같은 모달 동작을 지원하되, 직접 URL 진입에서는 같은 정보를 별도 2-depth 없이 보여준다.
- 게임에 없지만 서비스에 필요한 추가 정보는 같은 상세 표면 안의 접을 수 있는 정보 레일에 둔다. 댓글, 관련 리소스, 패치 이력, 내부 메타데이터처럼 탐색을 돕는 정보도 별도 페이지 전용 정보가 되면 안 된다.
- 정보 레일은 데스크톱에서 주 리소스 옆에, 모바일에서 주 리소스 아래에 배치한다.
- 정보 레일은 주 리소스보다 시각적으로 앞서면 안 되며, 정보량이 길어질 때는 같은 상세 표면 안에서 접힘, 탭, 섹션으로 정리한다.
- 상세 내 cross-reference는 hover preview와 클릭 이동을 제공한다.
- 긴 설명과 선택지는 게임 텍스트의 줄바꿈/강조를 살리되, 화면 폭에 맞게 읽히도록 한다.

## Rich Patch Notes

Rich 패치는 Steam 패치노트를 소스로 하되, 백과사전 데이터와 연결되는 서비스 콘텐츠다.

- 기본 게임 요소 강조는 gold다.
- hover 시 해당 카드/유물/포션/파워/몬스터/이벤트/고대의 존재를 렌더한다.
- 클릭 시 가능한 경우 해당 백과사전 리소스로 이동한다.
- 버프는 green + sine, 너프는 red + jitter를 사용한다.
- 특정 게임 요소 이름은 현재 PCK/백과사전 데이터의 공식 한국어명을 사용한다.

## 반응형 규칙

- 데스크톱: 좌측 필터를 열어 둔 상태가 기본이다.
- 모바일: 필터는 drawer로 열고, 본문 스크롤을 방해하지 않는다.
- 상단바 텍스트와 검색창은 한 줄 안에서 겹치지 않아야 한다.
- 카드/타일/아이콘 grid는 고정 폭 또는 안정적인 responsive track을 사용해 hover/로딩/카운트 표시가 레이아웃을 흔들지 않게 한다.

## 남은 디자인 과제

- 몬스터 인덱스를 게임처럼 긴 텍스트 목록으로 갈지, 현재처럼 이미지/이름 중심의 도감 목록으로 유지할지 결정이 필요하다.
- 몬스터 상세는 Spine 렌더가 배경에 떠 있는 듯한 비스티어리식 화면으로 옮기는 별도 시안이 필요하다.
- 정보 레일의 기본 접힘/펼침 상태는 전체 규칙으로 확정하지 않고, 카드/유물/포션/몬스터 같은 실제 리소스 종류별 mockup을 본 뒤 결정한다.
- 인카운터 조합은 `GenerateMonsters()` 분기와 확률/동등 선택 여부를 구조화하면 더 정확하게 보여줄 수 있다.
- Spine/VFX 표시 옵션은 미완성, mimic, static-only, 파싱 실패 상태를 토글 가능한 메타데이터로 분리한다.
